Designing with AI

Design Process

I can take a digital product from an initial sketch to a working, documented prototype while maintaining ownership of the design decisions.

Overview

AI has expanded my role from designing interfaces to shaping complete product experiences. I use AI as a design and engineering partner to move from discovery to a refined interface, working prototype, documented system, and developer-ready specification. My role remains active throughout: establishing the direction, defining constraints, evaluating recommendations, editing the design, reviewing the code, and deciding what belongs in the final product.

From sketch to product direction

To gain clarity, I'll sketch to work out interactions between iterative design passes

My process often begins away from the screen, with a sketch that captures the underlying interaction or information model. I use AI to examine the concept, identify missing states, challenge assumptions, and suggest ways the interface could behave. Those conversations help me turn an incomplete idea into a clearer product direction before investing time in high-fidelity design or code.

I may ask AI to translate the sketch into an initial wireframe or interface proposal, but I do not treat the first output as the solution. I compare it against the original intent, remove unnecessary complexity, refine the hierarchy, and redirect the work when the interface begins solving the wrong problem. This creates a rapid discovery loop in which ideas become visible and testable early.

Designing through conversation

Working effectively with an AI agent requires more than writing a single prompt. I use ChatGPT to establish the product context, explain what has already been decided, define constraints, and give the agent enough information to reason with me. The agent and I are a team. I then steer the conversation through targeted questions, corrections, comparisons, and increasingly specific decisions.

This process allows me to use AI for discovery, defining requirements, design critique, alternative exploration, state planning, accessibility considerations, technical feasibility, and architectural reasoning. I can ask the agent to defend a recommendation, expose tradeoffs, or reconsider an approach when it creates unnecessary complexity. The conversation becomes part of the design process—not a shortcut around it.

How I steer the work

  • Establish product context
  • Define clear constraints
  • Question recommendations
  • Compare alternatives
  • Protect design intent
  • Reduce unnecessary complexity

Prototyping in working code

I use VS Code to edit prototypes and perform code reviews.

I use Codex to translate approved design direction into working HTML, CSS, and JavaScript prototypes in Github branches. The code allows me to evaluate behavior that static screens cannot fully communicate, including responsive layouts, interactive states, panel transitions, tooltips, animations, and data visualizations. I monitor while the agent builds, steer if needed, review in the browser, test interactions, and refine the experience and code through iterative passes.

I do not hand the agent an open-ended request and wait for a finished product. I define the intended behavior, work within the existing architecture, review what changed, and steer the implementation when it adds too much code or departs from the design system. This paired-programming workflow gives me greater control over execution while helping me develop stronger technical judgment.

What I prototype

  • Responsive interfaces
  • Interactive behaviors
  • Component states
  • Panel transitions
  • Motion studies
  • Data visualizations
  • Internal design tools

Selecting a component will update the existing inspector without requiring the interface to be rebuilt. The goal is to make the relationships between visual design, interaction behavior, accessibility, and implementation easier to understand.

Specifications and developer handoff

Pushing bug fixes in Github

Once an experience has been validated, I use AI to help convert the decisions made during design into structured documentation. Depending on the product, this may include a product brief, feature specification, behavior specification, functional requirements, interaction notes, accessibility guidance, code examples, or implementation constraints. I review and revise these documents so they describe the approved experience rather than merely summarizing the agent’s output.

I also use this workflow to create project documentation, organize Markdown files, prepare Git branches, review changes, and maintain a record of why important decisions were made. This connects discovery, design, implementation, and handoff instead of allowing each phase to become disconnected from the last.

Handoff deliverables

  • Product briefs
  • Feature specifications
  • Behavior specifications
  • Functional requirements
  • Accessibility notes
  • Code examples
  • Implementation guidance

A human-directed AI workflow

AI increases the range of work I can execute, but it does not replace judgment or accountability. I remain responsible for determining what problem should be solved, whether a recommendation is appropriate, how the experience should feel, and when the work is ready to move forward. I validate the output through visual review, interaction testing, code inspection, iteration, and comparison against the established requirements. In some cases, I have decided not to accept an agent recommendation, instead making a few edits in code myself to solve the larger problem without adding additional complexity. Since I have a background in front-end development, I am also learning the style of how AI agents write code while I review the code myself, so I can make minor edits if necessary.

Result

AI does not make the product decisions for me. It gives me a multidisciplinary partner I can direct, question, and collaborate with as I move a product from concept to execution.

Reflection

This approach allows me to operate across product strategy, visual design, interaction design, prototyping, documentation, and early implementation. It is especially valuable for zero-to-one work, where the team needs someone who can bring structure to an unclear idea and steadily turn it into something concrete.

“Working with AI has made my design process more technical, more iterative, and more complete. I am no longer limited to showing how a product might work—I can prototype its behavior, examine its architecture, document its decisions, and prepare it for implementation.”

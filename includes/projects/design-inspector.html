Design inspector

Prototype

Exploring adaptive interfaces through a working design-system prototype.

Overview

Design Inspector is an independent interface exploration built with HTML, CSS, and JavaScript. It connects a polished conversational UI to the tokens, states, classes, and accessibility decisions supporting each component.

The project began as a working component inspector and is evolving into a broader system for examining multiple components inside a complete interface.

  • Designed a dashboard component highlighting current payment status and billing actions
  • Improved visual hierarchy for billing and payment actions
  • Strengthened UI consistency across payment states
  • Increased payment task completion rates over previous year

01. One system, multiple modes

Above: My original designer UI sketches

Each mode preserves the same structural foundation while adapting its theme, status, available actions, and interface content to the active context. The conversational interface uses one shared component system across four experience modes:

The four modes:

  • AI self-service
  • SMS continuation
  • Live support — Customer view
  • Live support — Agent workspace
Result

This approach demonstrates how a single design system can support different users and interaction types without creating separate products for every variation.

02. A working component inspector

Above: Original payment dashboard component

Unlike browser DevTools, which exposes raw implementation details, this inspector presents curated design intent: the system decisions that should remain visible to designers, developers, and anyone evaluating how the interface was constructed.

Prototype inspection categories:

  • Tokens
  • States
  • Classes
  • Accessibility
Result

Selecting a category updates the inspector with information relevant to quickly validate the intent of the active component.

03. From component to design inspection

Above: Original payment dashboard component

The next iteration expands the existing prototype from one isolated component to multiple inspectable components inside a complete interface.

Each conversation mode will contain intentionally defined inspection targets, such as:

  • conversation headers
  • message patterns
  • contextual actions
  • message composers
  • agent-assistance tools

Selecting a component will update the existing inspector without requiring the interface to be rebuilt. The goal is to make the relationships between visual design, interaction behavior, accessibility, and implementation easier to understand.

Result

Design Inspector explores how complex interface systems can be made visible, understandable, and reusable. It reflects my approach to emerging interface design: combining visual systems, interaction thinking, accessibility, and front-end implementation to turn complex product behavior into a clear and cohesive experience.

Reflection

Designing the payments experience reinforced how critical clarity and consistency are when users manage important tasks like insurance coverage and billing. When user interfaces clearly communicate status, next steps, and confirmation states, members are able to complete transactions with greater confidence and trust in the platform.

“Clear, system-driven interface design can turn complex account management tasks into confident, everyday actions.”

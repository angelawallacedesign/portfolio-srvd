# Angela Wallace Portfolio

This repository contains my personal portfolio and the ongoing evolution of the projects inside it.

Every project begins the same way: understanding the problem, sketching ideas, refining the interaction, and building toward a production-quality experience. Nothing in this portfolio is based on a purchased template. Every screen, layout, interaction, component, and visual system was designed from concept through implementation.

## Design Philosophy

For most of my career, I worked as a Visual Designer responsible for the final design work before engineering handoff. Working closely with front-end developers taught me that successful products are built through shared understanding—not simply handing over mockups.

My background in HTML and CSS gave me an advantage when designing interfaces because implementation constraints influenced design decisions from the beginning.

Today, AI has expanded what's possible.

Rather than using AI to generate one-off solutions, I wanted to understand how it reasons, how it structures code, where it succeeds, where it struggles, and how to guide it through iterative development. That meant learning far beyond prompting—writing structured context, defining guardrails, observing runtime behavior, debugging, refining implementations, and keeping large projects consistent across many iterations.

One of the hardest problems was maintaining continuity. Initial output is easy. Continuing development without losing architecture, introducing regressions, or overwriting previous work is much harder.

This portfolio is the result of solving that challenge.

## Pattern Inspector

One of the largest experiments in this repository is the Pattern Inspector.

It transforms interface components into inspectable design artifacts, exposing implementation details including design tokens, states, CSS classes, accessibility considerations, annotations, and documentation alongside the live interface.

The goal is to reduce ambiguity between design and engineering by making implementation decisions visible instead of hidden behind static mockups.

## Portfolio Architecture

The site itself follows a layered presentation system.

- **Frame / Matte Layout** creates consistent visual breathing room around every page.
- **Layer 3** establishes a reusable content stage that keeps layouts aligned while allowing each project to have its own identity.
- **Visualization components** communicate design thinking, systems, and process rather than serving as decorative graphics.

These patterns are intentionally reused throughout the portfolio to create consistency while remaining flexible enough for future projects.

## Why This Repository Exists

This repository represents more than a portfolio.

It documents an ongoing exploration of AI-assisted product design and front-end development—combining product thinking, visual design, interaction design, documentation, implementation, and engineering collaboration into a single workflow.

As expectations continue to expand across design and engineering, I'm interested in understanding where those disciplines meet and how AI can become a thoughtful engineering partner rather than simply a code generator.
